Here I will describe a slightly higher-level view on automatic differentiation, as an informal, introductory design spec for the library I'm writing here.


What does it mean to be "higher level" than classical autodiff. Well, in pytorch the basic operations are low-level: addition of tensors, elementwise multiplication, \(\tanh\), \(\sin\), for example.

For what I'll consider here, the basic primitive is a global constraint, which I will also call a block. Specifically, it has the form $$\op F(h, x, \theta) = 0$$ where \(h \in \hs H, x \in \hs X, \theta \in \hs \Theta\) are the block's state, inputs and parameters, respectively, and $$\op F : (\hs H, \hs X, \hs \Theta) \rightarrow \hs H,$$ i.e. it constraints the state variables. We require it to be locally well-posed, i.e., so that $$(\op D_h \op F(h, x, \theta))^{-1}$$ can be defined, i.e. the state Jacobian is non-singular, so we can apply the inverse function theorem at each permissable point where \(\op F(h, x, \theta) = 0\).

A key observation is that if we choose a loss function \(L : \hs H \rightarrow \R\), then the gradient descent flow is $$\delta \theta = -(\op D_\theta \op F)^* \cdot (\op D_h \op F)^{-*} (\nabla_h L)$$ So, the parameter and state Jacobian of \(\op F\) are the necessary linear operators required to do backpropagation on any loss. To make things nicer, we define some symbols: $$\text{ Generator: } \, \op J_h := \op D_h \op F; \, \text{ Propagator/Green's Operator: } \, \op P := \op J_h^{-1}$$ $$\text{ Parameter Jacobian: } \, \op J_\theta := \op D_h \op F; \, \text{ Parameter Core: } \, \op K :=(\op J_\theta)(\op J_\theta)^*$$ $$\text{ Error Signal: } \, \err(h) := \nabla_h L(h)$$ Using these operations and the implicit function theorem, we can see that the gradient flow in the state is first-order given by $$\delta h = -\op P \op K \op P^* (\err(h))$$ i.e., the operator \(\op P \op K \op P^*\) is the State-space Neural Tangent Kernel (sNTK) for the constraint \(\op F\).



Guiding Design Philosophy

Now, the whole design principle is this: provide fast methods for computing these operators by default, relying on fast matrix-free approaches, and specialize them using simplifying assumptions. For example, in a model with weights or one with high redundancy, there is a lot that can be done to make the Jacobians explicit. Finally, we allow the user to manually define any Jacobian by hand through hooks into the model, choose approximation methods to this Jacobian (e.g., for alternative learning rules), or specify additional structured simplifying assumptions on the model.





Jacobians Through VJP and JVP

First I will outline the default behavior for a generic global constraint block, which effectively derives the operators in a matrix-free way. There are three key things to implement: (1) \(\op J_h\) the Generator, (2) \(\op P = (\op J_h)^{-1}\) the Propagator and (3) \(\op J_\theta\) the Parameter Jacobian. This means that operators are generic interfaces providing a \(\mathrm{\_\_call\_\_}\) and \(\mathrm{adjoint\_call}\) functionality, for the operator \(M v, M^* p\), respectively. The key is that this totally relies on pytorch, I do not want to re-invent the wheel. Specifically, we can implement \(\op J_h, \op J_\theta\) using the \(\mathrm{vjp}\) and \(\mathrm{jvp}\) operations, which are fundamental to autodiff libraries (e.g., Jax and Pytorch). We also use \(\mathrm{vmap}\) to do such operations in a batched way, i.e. to implement operator \(\mathrm{matmul}\) and \(\mathrm{rmatmul}\).

Computing the Propagator/Green's Operator

Implementing \(\op P\) as a matrix-free inverse can potentially pull in a huge amount of numerical linear algebra (e.g., Krylov methods). The most simple geometric series (Neumann series), however, says that $$\op P = (\op J_h)^{-1} = \sum_{t=1}^\infty (\op I - \op J_h)^{t},$$ which can be viewed as integrating over all chains of jacobians of length \(t\). In general you only get convergence of this series when \(\rho(\op I - \op J_h)\) is below one. However, amazingly, this operator is typically nilpotent, i.e. there is some \(T\) with \((\op I - \op J_h)^T = 0\), so that this series converges, irregardless of \(\rho(\op I - \op J_h)\), hence why backproapgaton is provably correct even though it can cause exploding gradients. But, \(\op J_h\) is provably nilpotent for any directed acyclic graph (DAG), so this Neumann series approach is surprisingly effective. Indeed, it is the fundamental way in which backpropation typically takes place, forming all chains of Jacobians to approximate \(\op P^* v\) for \(v \in \hs H\) implicitly.

Below, I will talk about how \(\op P\) can be calculated in non-nilpotent settings, such as for PDEs, continuous time models or deep equilbrium models, but this is nonstandard for my own work. More practical for me, I will discuss how the propagator associated with compositions, sums and other operations on blocks can be automatically derived and how tricks such as preconditioned Neumann series, splitting and truncation could potentially be applied to make it faster or more stable. Almost all of these tricks are far more natural from this global operator perspective.



Operations on Blocks

Stacking Blocks

Consider stacking the state of one block, \(\op F_1(h_1, x, \theta)\) into another block, \(\op F_2((h_1, h_2), x, \theta)\), with identical input. We can expand the state-space and parameter space, creating a new block, $$\op{F_{stk}(\begin{pmatrix} h_1 \\ h_2 \end{pmatrix}, x, \begin{pmatrix} \theta \end{pmatrix}) = \begin{pmatrix} \op F_1(h_1, x, \theta_1) \\ \op F_2(h_2, x, \theta_2) \end{pmatrix}$$ We let \(h, \theta\) be the new states in these expanded spaces. We see that $$\op J_{\theta, \op {stk}} = \begin{pmatrix} \op J_{\theta,1} & 0 \\ 0 & \op J_{\theta,2} \end{pmatrix}$$ and $$\op J_{h,\op{stk}} = \begin{pmatrix} \op J_{h,1} & 0 \\ 0 & \op J_{h,2} \end{pmatrix}$$
